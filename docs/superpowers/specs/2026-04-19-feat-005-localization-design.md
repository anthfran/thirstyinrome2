# FEAT-005: Localize UI ‚Äî Design Spec

**Date:** 2026-04-19
**Feature:** FEAT-005

## Overview

Add a `Localizable.xcstrings` String Catalog to the app covering the 11 user-visible UI strings across 25 locales. Fountain names and the `"Fontanella"` fallback title are excluded ‚Äî they are proper names and remain as-is in all locales.

## Strings Inventory

| Key | File | Notes |
|---|---|---|
| `"Rome"` | ContentView | City name, varies by locale (Roma, Rom, etc.) |
| `"Cluster of %lld fountains"` | ContentView | Accessibility label; `%lld` is the Int format specifier for `cluster.count` |
| `"Zooms to this cluster"` | ContentView | Accessibility hint |
| `"Waiting for GPS signal\u{2026}"` | LocationButton | Toast; ellipsis is Unicode `U+2026` |
| `"My Location"` | LocationButton | Button label |
| `"Location Access Required"` | LocationButton | Alert title |
| `"Open Settings"` | LocationButton | Alert button |
| `"Cancel"` | LocationButton | Alert button; iOS also provides system translation, catalog takes precedence |
| `"To re-center on your position, enable Location in Settings."` | LocationButton | Alert message |
| `"Open in Apple Maps"` | FountainSheet | "Apple Maps" is a brand name and stays in English; only "Open in" translates |
| `"Open in Google Maps"` | FountainSheet | Same ‚Äî "Google Maps" stays in English |

## Locales

25 locales in total:

`en`, `it`, `de`, `fr`, `es`, `pt-BR`, `ja`, `zh-Hans`, `zh-Hant`, `nl`, `pl`, `ru`, `ko`, `sv`, `da`, `nb`, `cs`, `hu`, `ro`, `fi`, `ar`, `tr`, `el`, `hr`, `sk`

English (`en`) is the source language and the fallback for any missing translations.

## File Structure

One new file: `thirstyinrome/Localizable.xcstrings`

Placed directly in the app folder. Because the project uses File System Synchronized Groups (Xcode 16+), the file is auto-included in the build with no `project.pbxproj` edit.

## Code Changes

No Swift source changes required. SwiftUI's `Text("‚Ä¶")` and `Label("‚Ä¶")` already accept `LocalizedStringKey`, so they look up the catalog automatically for all string literals passed to them.

The interpolated accessibility label on ContentView line 46:
```swift
.accessibilityLabel("Cluster of \(cluster.count) fountains")
```
uses the `LocalizedStringKey` overload, which substitutes the `Int` as `%lld` when forming the lookup key. The catalog key `"Cluster of %lld fountains"` matches this automatically ‚Äî no code change needed.

## Translation Approach

Machine translations generated by Claude inline during implementation. Accepted per backlog AC ("machine translation acceptable for initial pass"). The `.xcstrings` JSON is written in full with all 25 locale entries for each string.

## Acceptance Criteria

- All 11 user-visible strings live in `Localizable.xcstrings`, not hardcoded in Swift files (fountain names and `"Fontanella"` are explicitly excluded)
- All 25 locales have translations in the catalog
- Device locale automatically selects the correct language; English is the fallback
- Build succeeds and all existing tests pass
